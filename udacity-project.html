Lesson 3 - CSS

Overview and Reminders
CSS code used to add style. Concept of avoiding repetition - helps make efficient code. HTML tells the browser where to find the style info - found in the head tag.

Example app html file and style.css files were demonstrated. We turned the description style text color to red to demonstrate. Scratchpad.io was shown as an alternative to demonstrate the css inside the same html doc.

Definitions
Definition of CSS was declared as Cascading Style Sheets. The C was the most important because of the cascading effect. A short demo showed how em was still pink but it makes sense because despite em being declared as pink, it was contained within an h1 that was declared to be pink.
Concepts
What does CSS mean then? In general Style – but really, rules. CSS is a bit of the reverse of our legal system where - National Laws, State Laws, Municipality Laws, and House Rules are applied. CSS most specific rules are applied to all elements! - This is what Cascading Means. This is powerful and efficient! CSS looks for the rule that best describes the element in question and it is then applied.
Characteristics
Inheritance was introduced and explains the pink example up above. The wiki (http://en.wikipedia.org/wiki/Cascading_Style_Sheets#Inheritance) states "... Descendant elements may inherit CSS property values from any ancestor element enclosing them ..." The distinction was made that inheritance applies to text but not box related properties. Specifically, these can be inherited, "color, font, letter-spacing, line-height, list-style, text-align, text-indent, text-transform, visibility, white-space and word-spacing.” Several others are also not inherited, including: "background, border, display, float and clear, height and width, margin, min- and max-height and -width, outline, overflow, padding, position, text-decoration, vertical-align and z-index." The inheritance property allows for much less code repetition!
Application Syntax and Reference
CSS is written and applied in the following formats: Selector: we tell it what elements it will apply to. In this case it could be the h1 tag – and will apply to each and every h1 tag. We could also write the style attribute:description and apply it to a defined class – much more control and granularity here. CSS reference documentation was introduced - https://developer.mozilla.org/en-US/docs/Web/CSS/Reference. And a quick intro article was introduced here - https://css-tricks.com/how-css-selectors-work/. Several font-style properties were reviewed as well as font-weight to make it look bold. We can place CSS in the head of the HTML, in the CSS file outside of HTML, or inline with the HTML which is undesirable because it leads to repetition which CSS was designed to avoid. Note: CSS comments begin with forward slash and asterisk and end opposite.
Demonstration
Demo 2 and Demo 3 highlighted the power of CSS. The HTML remained unchanged – only the CSS file was adjusted to render a more polished look. Use proper markup – using H1 instead of the div tag in this case. Search engines use these to index.
Box Attributes, Sizing, and Positioning
Positioning Boxes and the Model with 4 components: Margins – provides area around the border – space between boxes, Border – goes around padding and content, Padding – clears an area around the content, Content – the image or text shown (the nucleus) other layers protect it, Element Width = left border, left pad, content width, right padding, right border.

CSS style box-sizing border-box = calcs border and padding – just set width of overall box; padding and border remain the same. Boxes can be set in Pixel or Percentage. Percentage renders better as page size is adjusted. Max-width can also be set and must be less than 100% for flexbox. Positioning – the most complicated concept – so they don’t take the entire line. Flex box layout was introduced. Several elements or divs can be next to each other using flexbox in the CSS. Note: Use an ALT on image tags in case browser can’t show the image.

Development Process, Tools, and Error Correction
Code, Test, Refine was our process to build. Look for natural boxes. Look for repeated styles and semantic styles. Write my html. Apply styles from biggest to smallest. Fix and try to view in other browsers.

Dev Tools can be a big help during refinement – we can modify attributes on the fly.

Ensure HTML and CSS are written in compliance to the standards. Test your html, and css to ensure they are free of errors and in compliance.
